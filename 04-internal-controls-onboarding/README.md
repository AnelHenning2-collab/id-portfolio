# Internal Controls & Audit Basics — New Hire Onboarding — Case Study

**Format:** Self-paced onboarding module (~7 minutes)
**Audience:** New hires joining a GRC/audit-adjacent team with no prior audit background
**Role:** Instructional Designer (needs assessment through build)

## 1. Needs Assessment

**The gap:** New hires joining compliance or audit-support teams often arrive with strong writing or research skills but no working vocabulary for the audit process itself — terms like "control," "ITGC," or "evidence request" get used in their first team meeting with no explanation, leaving them guessing at context for weeks.

**Why this matters:** A new hire who understands the basic audit cycle (what a control is, what evidence looks like, what a finding means) can contribute to their first evidence-gathering task in week one instead of week four, and asks better clarifying questions instead of guessing.

**Constraint:** Onboarding time is limited and competes with account setup, HR paperwork, and team introductions — this needs to be short enough to fit into a first-day schedule without needing a dedicated training block.

## 2. Learning Objectives

By the end of this module, learners will be able to:
1. Define "internal control" in plain language and give an example
2. Describe the basic audit cycle: control identified → evidence requested → evidence reviewed → finding documented
3. Explain the difference between a "finding" and a "recommendation"
4. Identify, given a sample request, what would count as acceptable evidence

## 3. Design Decisions

- **Onboarding tone, not lecture tone** — written as if a helpful teammate is walking a new hire through it, not as a formal policy document.
- **Cycle framing, not glossary framing** — terms are introduced in the order they'd actually be encountered during a real audit engagement, not alphabetically or by importance, so the sequence itself teaches the process.
- **A worked example throughout** — one running example (checking whether terminated employees lose system access on time) is used across every screen, so the abstract cycle stays anchored to something concrete.

## 6. Evaluation Plan

- **Level 1:** End-of-module reaction question
- **Level 2:** Scored knowledge check
- **Level 3:** New hire's manager reviews their first real evidence-gathering assignment against the concepts from this module within the first two weeks
- **Level 4:** Reduced onboarding ramp-up time to first independent evidence request, tracked across new hire cohorts
