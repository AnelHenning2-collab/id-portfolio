# Phishing & Social Engineering Awareness — Microlearning Refresher — Case Study

**Format:** Microlearning — 3 screens, under 3 minutes
**Audience:** All-staff annual refresher (any department, any technical level)
**Role:** Instructional Designer (needs assessment through build)

## 1. Needs Assessment

**The gap:** Annual phishing training often gets skipped or clicked through without attention because it's too long and repeats material people already half-remember. A refresher needs to respect people's time or it won't get engagement at all.

**Why this matters:** Phishing remains one of the most common ways organizations get breached. A refresher that's actually completed and remembered is more valuable than a comprehensive course that gets abandoned halfway through.

**Constraint:** This is deliberately the shortest module in this portfolio — a genuine test of designing for minimum viable attention span, not a shorter version of a long module.

## 2. Learning Objectives

By the end of this module, learners will be able to:
1. Identify two visual red flags common in phishing emails
2. Correctly choose the safe response when they suspect a phishing attempt

## 3. Design Decisions

- **Three screens, no more.** This is a deliberate constraint, not a shortcut — real microlearning is designed to be short from the start, not a trimmed-down long module.
- **Spot-the-difference format** — learners compare a real-looking phishing email against red-flag callouts, since recognition (not recall of a rule) is the actual skill needed in the moment an email lands in their inbox.
- **Single call to action** — the module ends with exactly one instruction to remember, not a list, because a refresher this short only has room for one thing to stick.

## 6. Evaluation Plan

- **Level 1:** Completion rate itself is the primary signal for a microlearning refresher — if people finish it, the format worked
- **Level 2:** Single knowledge check embedded in the module
- **Level 3/4:** Click-rate on the organization's next simulated phishing test, compared to the test run before this refresher went out
